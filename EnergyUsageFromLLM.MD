## Energy estimated Usage of an LLM

Formula:

EKH = ((Tin + Tout) * Ej * C(Q)) / 3600

### Variables

**Tin**  
Number of input tokens in the user prompt.

**Tout**  
Number of output tokens generated by the language model.

**Ej**  
Energy consumption per processed token in **Joules (J)**.  
Modern AI accelerators such as GPUs require a certain amount of energy for each token processed during inference. For example, GPUs like the NVIDIA H100 are estimated to consume roughly **0.4 Joules per token**, depending on workload, batch size, and system configuration.

**C(Q)**  
Model complexity factor based on the **number of parameters of the trained model**.  
Example: A model with **70 billion parameters (70B)** requires significantly more computation than smaller models.

**3600**  
Conversion factor from **Joules (J) to Watt-hours (Wh)**.

### Explanation

The total energy consumption of a single LLM inference request depends mainly on three factors:

1. **Token count** – the total number of processed tokens (input + output).  
2. **Hardware efficiency** – the energy required by GPUs or accelerators to process tokens.  
3. **Model size / complexity** – larger models with more parameters require more computational operations.

This simplified formula provides an approximate estimation of the energy consumption of a single inference request.
